# Architecture Plan: Task CRUD API

**Feature**: Task CRUD API (Phase II, Cycle 1)
**Created**: 2026-03-31
**Status**: Draft

## Scope

### In Scope
- FastAPI application scaffold with project structure
- SQLModel Task model with Neon PostgreSQL connection
- 6 REST endpoints (CRUD + toggle + list with filter)
- Pydantic request/response models
- CORS configuration
- Health check endpoint
- pytest test suite

### Out of Scope
- Authentication/JWT (Cycle 3)
- Frontend (Cycle 2)
- Alembic migrations (use create_all for initial setup, migrate to Alembic in Phase III)
- Deployment

### Component Design

#### 1. Database Layer (`db.py`)
- `create_engine()` with `DATABASE_URL` from env
- `pool_pre_ping=True` for Neon cold start handling
- `get_session()` generator for FastAPI dependency injection
- `init_db()` function that calls `SQLModel.metadata.create_all(engine)` for initial table creation

#### 2. Models (`models.py`)
```
TaskBase(SQLModel)           ‚Üê Shared fields: title, description
  ‚îú‚îÄ‚îÄ TaskCreate(TaskBase)   ‚Üê POST body (title required)
  ‚îú‚îÄ‚îÄ TaskUpdate(SQLModel)   ‚Üê PUT body (all optional)
  ‚îú‚îÄ‚îÄ Task(TaskBase, table)  ‚Üê DB table: id, user_id, completed, timestamps
  ‚îî‚îÄ‚îÄ TaskResponse(TaskBase) ‚Üê API response: id, user_id, completed, timestamps
```

#### 3. Routes (`routes/tasks.py`)
| Method | Path | Handler | Returns |
|--------|------|---------|---------|
| GET | `/api/{user_id}/tasks` | `list_tasks(user_id, status?)` | `list[TaskResponse]` |
| POST | `/api/{user_id}/tasks` | `create_task(user_id, body)` | `TaskResponse` (201) |
| GET | `/api/{user_id}/tasks/{task_id}` | `get_task(user_id, task_id)` | `TaskResponse` |
| PUT | `/api/{user_id}/tasks/{task_id}` | `update_task(user_id, task_id, body)` | `TaskResponse` |
| DELETE | `/api/{user_id}/tasks/{task_id}` | `delete_task(user_id, task_id)` | `{"ok": true}` |
| PATCH | `/api/{user_id}/tasks/{task_id}/complete` | `toggle_complete(user_id, task_id)` | `TaskResponse` |

#### 4. App Entry (`main.py`)
- Create FastAPI app with title, version
- Add CORS middleware (allow frontend origin from env)
- Include tasks router
- Health check at `/health`
- On startup: call `init_db()` to create tables

### Data Flow
```
Client Request
  ‚Üí FastAPI Router (routes/tasks.py)
    ‚Üí Pydantic validation (automatic)
      ‚Üí Session dependency (db.py)
        ‚Üí SQLModel query (models.py)
          ‚Üí Neon PostgreSQL
        ‚Üê Task row
      ‚Üê Task object
    ‚Üê Pydantic response serialization
  ‚Üê JSON Response
```

### Error Handling
| Scenario | Status | Detail |
|----------|--------|--------|
| Validation fails (Pydantic) | 422 | Auto-generated by FastAPI |
| Task not found | 404 | "Task not found" |
| Empty title after strip | 422 | Pydantic min_length=1 |
| Title > 200 chars | 422 | Pydantic max_length=200 |
| Description > 1000 chars | 422 | Pydantic max_length=1000 |
| DB connection error | 500 | FastAPI default handler |

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Sync vs Async DB | Sync (Session) | SQLModel Session is sync; simpler for Phase II. Can upgrade to async in Phase V if needed. |
| create_all vs Alembic | create_all on startup | Phase II is initial table creation; Alembic added in Phase III when schema evolves. |
| Task ID type | Auto-increment int | Simple, standard, sufficient for single-DB setup. |
| user_id source | URL path param (for now) | JWT-based user_id comes in Cycle 3; for now, trust path param. |

## Risks

1. **Neon cold start latency** ‚Äî mitigated with `pool_pre_ping=True`; first request may be slow (~1s)
2. **No auth yet** ‚Äî any user_id in URL is trusted; acceptable for Cycle 1, fixed in Cycle 3
3. **create_all in prod** ‚Äî acceptable for Phase II initial setup; must switch to Alembic before Phase III schema changes
